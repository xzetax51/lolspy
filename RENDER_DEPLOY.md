# Деплой на Render

## Что изменилось в коде

- `main.py` теперь сам поднимает Telegram-бота в фоне (через
  `@app.on_event("startup")`), используя переменные окружения
  `BOT_TOKEN` и `WEBAPP_URL`. Отдельный `bot.py` запускать не нужно —
  на Render достаточно ОДНОГО Web Service.
- Главная страница `/` отдаёт `static/index.html`, остальные файлы
  фронтенда лежат под `/static/...`.

## Шаги

### 1. Залейте проект на GitHub

Структура репозитория:
```
main.py
requirements.txt
render.yaml          (опционально, для автонастройки)
static/
  index.html
```

`bot.py` можно удалить — он больше не нужен (логика бота встроена в main.py).

### 2. Создайте сервис на Render

1. Зайдите на https://render.com → New → Web Service
2. Подключите ваш GitHub-репозиторий
3. Настройки:
   - **Runtime**: Python 3
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `uvicorn main:app --host 0.0.0.0 --port $PORT`
   - **Plan**: Free (для теста)

Если в репозитории лежит `render.yaml` — Render может подхватить
настройки автоматически (Blueprint).

### 3. Переменные окружения

В разделе **Environment** добавьте:

- `BOT_TOKEN` — токен от @BotFather
- `WEBAPP_URL` — будет URL вашего сервиса, например
  `https://lol-spy-bot.onrender.com`
  (узнать точный адрес можно после первого деплоя — Render покажет
  его в карточке сервиса; затем впишите его в переменную и
  передеплойте/перезапустите сервис)

### 4. Деплой

Нажмите **Create Web Service** (или **Deploy**). Render соберёт и
запустит проект. В логах должно появиться:

```
Telegram-бот запущен в фоне (polling)
INFO: Application startup complete.
```

### 5. Проверка

1. Откройте `https://ваш-сервис.onrender.com` в браузере — должна
   открыться страница игры (можно потыкать кнопки, юзер будет
   "web_xxxxx").
2. В Telegram напишите боту `/start` — должна прийти кнопка
   «🎮 Открыть игру», открывающая то же приложение внутри Telegram.

## Важные замечания про бесплатный план Render

- **Засыпание**: бесплатный Web Service "засыпает" после ~15 минут
  без HTTP-запросов и просыпается ~30-60 сек при следующем запросе.
  Поскольку бот держит постоянное polling-соединение с Telegram API
  изнутри процесса, это частично спасает от засыпания, но не
  гарантированно — Render всё равно может остановить инстанс.
  Для надёжной работы 24/7 в будущем стоит перейти на платный план
  или настроить внешний "пинг" сервиса (например, UptimeRobot,
  каждые 10 минут дёргать `/`).
- **In-memory хранилище**: при засыпании/перезапуске все активные
  лобби теряются. Для долгоживущих игр стоит подключить Redis
  (Render предоставляет Redis как отдельный сервис).
- Если в репозитории не указан `BOT_TOKEN`/`WEBAPP_URL`, сервис всё
  равно запустится (просто без бота) — это удобно, чтобы сначала
  проверить веб-часть, узнать домен, а потом вписать переменные.
